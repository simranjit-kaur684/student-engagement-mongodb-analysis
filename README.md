# Student Engagement Analytics using MongoDB

This project analyses student interaction behaviour within a Virtual Learning Environment (VLE) using the Open University Learning Analytics Dataset (OULAD). The objective is to identify engagement patterns, behavioural trends, and potential indicators of academic risk through MongoDB aggregation pipelines and visual analytics.

## Project Overview

The study focuses on analysing large-scale clickstream interaction data generated by students using the VLE. MongoDB is used as the primary data platform due to its flexible document-based structure and powerful aggregation framework, which enables efficient processing of high-volume behavioural datasets.

The project implements a MongoDB-based analytical pipeline to transform raw interaction logs into meaningful engagement insights.

## Dataset

The analysis uses the **Open University Learning Analytics Dataset (OULAD)** which contains anonymised student interaction data across multiple modules.

Key dataset components include:

- `studentInfo.csv` – student demographic and outcome information  
- `studentVle.csv` – student interaction logs with VLE resources  
- `vle.csv` – metadata describing learning resources  

The dataset contains **over 10 million interaction records**, making it suitable for big data analytics tasks.

## System Architecture

The project uses the following technologies:

- **MongoDB Atlas** – cloud database for storing interaction data  
- **PyMongo** – Python interface for MongoDB operations  
- **Pandas** – data preprocessing and ETL tasks  
- **MongoDB Aggregation Framework** – behavioural data analysis  
- **MongoDB Charts / Python Visualisations** – dashboard visualisation  

## Key Analytical Pipelines

The project implements several MongoDB aggregation pipelines:

1. **Temporal Engagement Analysis**  
   Identifies engagement trends across the course timeline.

2. **Engagement by Education Level**  
   Compares interaction levels across student academic backgrounds.

3. **Behavioural Segmentation**  
   Groups students into engagement tiers (At-Risk, Low, Moderate, High, Power Users).

4. **Resource Interaction Analysis**  
   Determines which VLE resources generate the highest student interaction.

## Visual Dashboard

A visual analytics dashboard was created using MongoDB Charts to communicate the analytical findings.  
The dashboard presents:

- engagement trends over time  
- demographic interaction comparisons  
- behavioural segmentation of students  
- resource interaction patterns  

These visualisations support the identification of engagement patterns and potential academic risk indicators.

## Repository Contents

## Purpose

The project demonstrates how NoSQL technologies and learning analytics can be combined to analyse large-scale educational data and generate actionable insights for improving student engagement and academic support.
